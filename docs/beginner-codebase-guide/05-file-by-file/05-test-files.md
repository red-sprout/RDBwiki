# 테스트 파일

## 포함된 파일 목록

현재 저장소에서 테스트 파일은 확인되지 않습니다.

## 이 파일 묶음의 역할

현재는 실제 테스트 코드 묶음이 없습니다. 이 문서는 테스트 부재와 추가하기 좋은 대상을 설명합니다.

## 전체 연결 관계

테스트를 추가하면 다음 구조가 됩니다.

`package.json`  
→ 테스트 script  
→ `lib/*.test.ts`  
→ `lib/dbms-filter.ts`, `lib/toc.ts`, `components/docs/SqlCodeBlock.tsx`

## 파일별 상세 설명

## `package.json`

### 이 파일의 역할

현재 사용 가능한 검사 명령어는 `lint`입니다.

### 이 파일이 필요한 이유

테스트 script를 추가하려면 이 파일을 수정합니다.

### 이 파일과 연결된 다른 파일

미래의 테스트 설정 파일입니다. 현재 코드에서 확인되지 않습니다.

### 핵심 코드 블록

```json
"lint": "eslint ."
```

### 코드 블록별 해설

ESLint는 정적 검사입니다. 동작 검증 테스트는 아닙니다.

### 이 파일에서 사용된 언어 문법

JSON입니다.

### 이 파일에서 사용된 프레임워크/라이브러리 기능

ESLint입니다.

### 초심자가 수정할 수 있는 부분

테스트 도구 설치 후 `test` script를 추가할 수 있습니다.

### 수정 전 코드

```json
"lint": "eslint ."
```

### 수정 후 코드

```json
"lint": "eslint .",
"test": "vitest"
```

### 수정 시 영향받는 파일

`package-lock.json`과 테스트 설정 파일이 함께 바뀔 수 있습니다.

### 이 파일을 이해한 뒤 알아야 하는 것

현재 `npm test`는 준비되어 있지 않습니다.
